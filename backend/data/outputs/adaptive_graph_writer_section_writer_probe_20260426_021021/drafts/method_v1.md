We introduce Adaptive Graph Writer, a section writer agent that bridges structured scientific writing with real visual generation. Unlike conventional agents that insert placeholder references (e.g., 'Figure X'), our method enables the section writer to *materialize* required visuals on demand using draw.io MCP — a deterministic, scriptable diagramming interface integrated into the workflow. This is grounded in a contract-driven architecture: each section is governed by a formal contract specifying not only required claims and citations but also explicit visual obligations — including artifact ID, narrative role, placement hint, and explanation requirement.

[FIG:fig_method_pipeline]

The pipeline (Figure 1) consists of three tightly coupled phases: (1) **Contract Activation**, where the section writer receives its execution contract containing source story fields (e.g., `core_idea`, `method_summary`) and visual obligations; (2) **Adaptive Generation**, where the writer composes narrative text *and*, when a required visual lacks a materialized artifact, invokes draw.io MCP tools to generate it — respecting layout semantics, labeling conventions, and narrative alignment; and (3) **Obligation-Aware Rendering**, where the final draft embeds both prose and inline visual references with traceable provenance. Critically, visual generation is not post-hoc or manual: it is triggered automatically during drafting when `visual_obligations.explanation_required == true` and `materialization_status == "planned"`, ensuring figures are authored *in context*, not inserted as afterthoughts.
